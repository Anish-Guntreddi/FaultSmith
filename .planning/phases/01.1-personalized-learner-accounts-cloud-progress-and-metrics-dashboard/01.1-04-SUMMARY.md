## Accomplishments

- `src/components/progress-sync.tsx` adds a compact non-blocking account surface inside My Progress only (never before the roadmap): Continue as guest, Create account, Log in, and Continue with Google, with accessible email/password forms (autocomplete email/new-password/current-password, show/hide with aria-pressed, confirm-password only during creation), Firebase password-policy checks, verification-pending with resend cooldown, generic reset confirmation, and bounded live status/alert text for every state.
- The sync state machine models `On this device`, `Creating account`, `Verification required`, `Signing in`, `Syncing`, `Synced to account`, and `Saved on this device—cloud unavailable` with retry; the dashboard badge and panel chip render the same bounded labels, and every failure path visibly degrades to local safety.
- Verified sign-in fetches a fresh ID token at request time only, reads the strict `progressSnapshotSchema` DTO, and displays `mergeCloudLearnerProfiles(local, cloud)` from React state — cloud-derived records never enter device storage, sign-out returns exactly to guest data, and the new outcome-identity dedupe (mirroring the server's SHA-256 idempotency material) prevents duplicated attempts and false provenance.
- Bounded local import is offered exactly once when the account has never imported (`localImportCompleted` false) and local history exists; 409 replay collapses to completed. Cloud-data deletion requires explicit confirmation and reports real failure; account deletion is offered only after data deletion, requires recent authentication (`recent_login_required` surfaced distinctly), and the adapter gained `refreshVerificationStatus`, `deleteCloudAccount`, and a test-gated `isProviderLinkingSupported` capability (default unsupported → safe existing-provider guidance, no silent merge or cross-UID copy).
- Assessment submissions attach the fresh token but the deterministic report and local progress update first; the bounded `cloudSync` fact then only updates the sync surface (silent refetch on `cloud_saved`, visible degraded state on `cloud_unavailable`).
- `next.config.ts` widens CSP only when cloud sync is configured, by exact origins proven empirically against the running popup flow: `https://identitytoolkit.googleapis.com` + `https://securetoken.googleapis.com` (connect-src), the regex-validated project auth domain (frame-src), and `https://apis.google.com` (script-src, gapi loader — without it the popup fails closed on a CSP violation before opening). No wildcard exists; the loopback-validated emulator origin joins only in development; the production `unsafe-inline` residual is documented in place; cloud-off production CSP and COOP are byte-identical to the prior baseline (verified by `smoke:production` header inspection).
- `playwright.firebase.config.ts` + `npm run test:e2e:firebase` drive the real browser bundle against the `demo-faultsmith` Auth/Firestore emulators only (fake API key, loopback-validated `connectAuthEmulator`, Admin emulator hosts); 15 browser tests cover password policy/confirm mismatch, unverified local continuity, verification refresh via emulator OOB links, resend cooldown, enumeration-resistant login and reset, Google popup success/cancellation/popup-block, one-time import provenance, signed refresh restoration, sign-out, deletion failure/success plus recent-auth account deletion, quota/timeout-shaped degradation with retry, verified-report-survives-persistence-failure, a persisted-state leak check (no email/password/token/uid/provider material in FaultSmith localStorage), and axe/password-manager/focus/1440x900/390x844 coverage.
- Cloud-off rollback is proven in the default suite: with Firebase unset the account surface offers no sign-in path, no login wall exists anywhere, the dashboard renders the local baseline, and zero requests reach any Firebase/Google origin.

## Decisions Made

- Cloud-merged progress is presented from React state only; FaultSmith localStorage keeps device-only progress plus a single identity-free opt-in boolean, making "sign-out returns to guest data" structurally true and keeping shared machines clean.
- Attempt dedupe uses a bounded outcome identity (excluding identifiers, timestamps, provenance) that mirrors the server's SHA-256 idempotency material, so the local random-id copy and the server derived-id copy of the same attempt collapse with `server_verified` preferred.
- Provider linking remains capability-gated off (`NEXT_PUBLIC_FAULTSMITH_PROVIDER_LINKING`) until Plan 06 real-provider proof; the UI shows safe existing-method guidance for collisions instead.
- COOP conditionally relaxes to `same-origin-allow-popups` only in cloud-configured builds — required for the popup opener relationship — while `smoke:production` continues asserting the strict cloud-off baseline.
