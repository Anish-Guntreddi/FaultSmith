## Accomplishments

- `attemptSummarySchema` accepts only approved lesson/project/skill/difficulty/source/status combinations validated against the existing lesson registry, with bounded score dimensions, hints, test runs, changed lines, a duration bucket, a bounded completion timestamp, an idempotency identifier, and `server_verified | local_import` provenance — no source, hypothesis, explanation, hint text, test output, provider data, identity token, or credential field can pass validation
- Deterministic v1-to-v2 migration preserves existing local learning progress verbatim and never fabricates attempt history; tampered containers, unknown keys, prose fields, unknown lessons, duplicates, and over-range values are all excluded
- Merge helpers provide monotonic per-lesson union, duplicate/idempotency replacement, and 50-attempt retention — cloud-ready and symmetric for distinct timestamps
- Metric selectors return honest nulls for empty and unverified-only states, average only server-verified attempts, and treat test iteration as descriptive process evidence, never a penalty
- My Progress renders inside the existing configure experience with the `On this device` storage label, 0/9 and per-phase progress, score-dimension bars, independent-solve and test-run evidence cards, strongest/reinforcement skills, recent attempts, a reasoned next-lesson launch button, and explicit practice-not-certification language; Guided Roadmap stays the default and lesson 1 remains reachable
- 43 domain unit tests plus a 5-scenario Playwright/axe suite (migration, tamper exclusion, verified-only dashboard updates, failing non-completion, refresh restoration, keyboard access, 1440x900 and 390x844 overflow) all pass alongside the full existing suites, production build, and client-bundle leakage scan

## Decisions Made

- The v1 `faultsmith:learning-progress:v1` key remains the roadmap's storage; the v2 profile composes it with the new dedicated `faultsmith:attempt-history:v1` key, so existing users lose nothing and no history is invented
- Certified metrics require both `status: "verified"` and `provenance: "server_verified"`; migrated/local-import data counts toward roadmap progress but never toward verified score averages
- Guided attempt derivation falls back to a catalog-style (lesson-free) summary when the lesson/challenge combination cannot be registry-verified, preferring safe partial evidence over dropped or inconsistent records
