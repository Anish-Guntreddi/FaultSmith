# Phase 01.1 — Validation Strategy

> Credential-free-first validation contract for personal metrics, optional Firebase identity/cloud progress, and rollback safety.

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | Vitest 4, Firebase Authentication/Firestore Emulator Suite, Playwright 1.61 + axe, Next.js production server |
| **Config file** | `vitest.config.mts`, `playwright.config.ts`, planned `firebase.json` and `firestore.rules` |
| **Quick run command** | `npm run lint && npm run typecheck && npm test` |
| **Full suite command** | `npm run security:source && npm run quality && npm run test:firebase && npm audit --audit-level=moderate` |
| **Estimated runtime** | focused under 30 seconds; full local/emulator target under 3 minutes |

## Sampling Rate

- **After every task commit:** Run focused changed Vitest files, then `npm run lint && npm run typecheck`.
- **After every plan wave:** Run `npm run security:source && npm test && npm run build`; add `npm run test:firebase` after the emulator foundation exists.
- **Before `$gsd-verify-work`:** Full suite, fallback/production smoke, bundle scan, dependency audit, guest primary demo, and independent QA/security review must be green on one SHA.
- **Before real Firebase configuration:** Full credential-free/emulator candidate must be frozen and green.
- **Before Netlify preview:** Real local Firebase guest/sign-in/sync/config-off smoke must be green; deployment still requires explicit approval.
- **Max feedback latency:** 30 seconds focused, 180 seconds full/emulator; external provider/deployment proof is recorded separately.
